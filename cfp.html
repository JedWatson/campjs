We're looking for short talks + workshops on cutting-edge client and serverside technology.

Topics we're interested in:

  • ES6/7/8
  • Web Components & other new/emerging web standards.
  • Browser Internals (e.g. rendering).
  • Node/Libuv/V8 Internals
  • Modular Application Design i.e. not frameworks.
  • Alternative Databases
  • Taking JavaScript to new environments (e.g. robots, native apps, operating systems)
  • Core Node (streams, tcp, etc) i.e. not frameworks.
  • JavaScript-powered Games (front/backend)
  • Original Modules or Research
  • Shell & editor tooling

We want to experiment with doing 'pair' presentations, i.e. every presentation is run by at least 2 people. If this is not possible, we can be flexible.

Talks

Limited to 20 minutes, minimum 2 presenters. We want talks to be well-prepared, focussed and streamlined. If you can't fit your content in 20 minutes, perhaps you should consider running a workshop.

Workshops

We want to run more workshops at CampJS. Workshops can run up to 90 minutes. Workshops are limited to 20 people at a time, and should repeat at least once. Minimum 1 tutor per 7 people. Tutors are only there to guide and answer questions. Workshops should be totally self-directed, automated, and available online for anyone to participate in.

We recommend use of the locally-grown workshopper framework. If workshopper doesn't do what you need, please consider submitting a pull request to improve it.

Sound good? Get the conversation started by submitting a proposal: Simply email us with a short outline on what you'd like to present!
